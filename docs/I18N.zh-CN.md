# 国际化 (i18n)

## 概述

本项目使用 [i18next](https://www.i18next.com/) 实现国际化，支持多语言自动检测和持久化。

## 支持的语言

- **英文** (`en`) - 默认回退语言
- **简体中文** (`zh`) - 中文

## 架构

```
src/i18n/
├── index.ts           # i18n 配置和初始化
└── locales/
    ├── en.json        # 英文翻译
    └── zh.json        # 中文翻译
```

## 配置

### 检测策略

语言按以下顺序检测：

1. **localStorage** - 之前保存的用户偏好
2. **navigator** - 浏览器/系统语言

### 关键设置

```typescript
{
  fallbackLng: "en",           // 默认语言
  interpolation: {
    escapeValue: false         // React 已经转义
  },
  detection: {
    order: ["localStorage", "navigator"],
    caches: ["localStorage"]   // 持久化用户选择
  }
}
```

## 使用方法

### 应用初始化

应用会在 `src/main.tsx` 中统一初始化 i18n：

```typescript
import "./i18n";
```

### 推荐 Hook

项目内提供了一个包装 Hook：

```typescript
import { useAppTranslation } from "@/hooks/use-app-translation";

function MyComponent() {
  const { t } = useAppTranslation();

  return <h1>{t("app.title")}</h1>;
}
```

`useAppTranslation()` 会：

- 返回 `t`
- 监听 `language-changed` 事件，保持多个 Tauri 窗口之间的语言同步

### 基础翻译

```typescript
import { useAppTranslation } from "@/hooks/use-app-translation";

function MyComponent() {
  const { t } = useAppTranslation();

  return <h1>{t("app.title")}</h1>;
}
```

### 带插值

```typescript
// 翻译文件
{
  "updater": {
    "versionAvailable": "版本 {{version}} 可用。"
  }
}

// 组件
const message = t("updater.versionAvailable", { version: "1.2.0" });
// 输出: "版本 1.2.0 可用。"
```

### 语言切换

```typescript
import { useTranslation } from "react-i18next";

function LanguageToggle() {
  const { i18n } = useTranslation();

  const toggleLanguage = () => {
    const newLang = i18n.language === "en" ? "zh" : "en";
    i18n.changeLanguage(newLang);
  };

  return <button onClick={toggleLanguage}>切换语言</button>;
}
```

只有在需要使用 `i18n.changeLanguage()` 这类方法时，才直接使用 `useTranslation()`。

## 添加新语言

### 1. 创建翻译文件

```bash
# 创建新的语言文件
touch src/i18n/locales/ja.json
```

### 2. 添加翻译

```json
{
  "app": {
    "title": "Tauriアプリテンプレート",
    "welcome": "Tauri + Reactへようこそ"
  }
}
```

### 3. 在配置中注册

```typescript
// src/i18n/index.ts
import ja from "./locales/ja.json";

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    zh: { translation: zh },
    ja: { translation: ja }, // 添加新语言
  },
});
```

## 最佳实践

### 1. 命名空间组织

按功能分组相关翻译：

```json
{
  "auth": {
    "login": "...",
    "logout": "..."
  },
  "profile": {
    "edit": "...",
    "save": "..."
  }
}
```

### 2. 一致的键命名

- 使用 **camelCase** 命名键：`checkForUpdates`
- 使用 **点号** 表示嵌套：`updater.checkForUpdates`
- 保持键名描述性但简洁

### 3. 避免硬编码字符串

❌ **错误：**

```typescript
<button>检查更新</button>
```

✅ **正确：**

```typescript
<button>{t("updater.checkForUpdates")}</button>
```

### 4. 保持一致性

确保所有语言文件具有相同的结构：

```bash
# 检查缺失的键
diff <(jq -S 'keys' src/i18n/locales/en.json) \
     <(jq -S 'keys' src/i18n/locales/zh.json)
```

## 测试

### 测试翻译键是否存在

```typescript
import { renderHook } from "@testing-library/react";
import { useTranslation } from "react-i18next";

test("翻译键存在", () => {
  const { result } = renderHook(() => useTranslation());
  expect(result.current.t("app.title")).not.toBe("app.title");
});
```

## 故障排查

### 语言未切换

1. 检查 localStorage：`localStorage.getItem("i18nextLng")`
2. 清除缓存：`localStorage.removeItem("i18nextLng")`
3. 验证语言代码与语言文件名匹配

### 翻译缺失

1. 检查控制台警告
2. 验证键路径正确
3. 确保回退语言有该键

### 插值不工作

```typescript
// ❌ 错误
t("message", "value");

// ✅ 正确
t("message", { variable: "value" });
```
