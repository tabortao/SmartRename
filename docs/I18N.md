# Internationalization (i18n)

## Overview

This project uses [i18next](https://www.i18next.com/) for internationalization, supporting multiple languages with automatic detection and persistence.

## Supported Languages

- **English** (`en`) - Default fallback language
- **Chinese Simplified** (`zh`) - 简体中文

## Architecture

```
src/i18n/
├── index.ts           # i18n configuration and initialization
└── locales/
    ├── en.json        # English translations
    └── zh.json        # Chinese translations
```

## Configuration

### Detection Strategy

Language is detected in the following order:

1. **localStorage** - Previously saved user preference
2. **navigator** - Browser/system language

### Key Settings

```typescript
{
  fallbackLng: "en",           // Default language
  interpolation: {
    escapeValue: false         // React already escapes
  },
  detection: {
    order: ["localStorage", "navigator"],
    caches: ["localStorage"]   // Persist user choice
  }
}
```

## Usage

### App Initialization

The app initializes i18n once in `src/main.tsx`:

```typescript
import "./i18n";
```

### Recommended Hook

This project provides a small wrapper hook for page/components:

```typescript
import { useAppTranslation } from "@/hooks/use-app-translation";

function MyComponent() {
  const { t } = useAppTranslation();

  return <h1>{t("app.title")}</h1>;
}
```

`useAppTranslation()`:

- returns `t`
- keeps language in sync across Tauri windows by listening to the `language-changed` event

### Basic Translation

```typescript
import { useAppTranslation } from "@/hooks/use-app-translation";

function MyComponent() {
  const { t } = useAppTranslation();

  return <h1>{t("app.title")}</h1>;
}
```

### With Interpolation

```typescript
// Translation file
{
  "updater": {
    "versionAvailable": "Version {{version}} is available."
  }
}

// Component
const message = t("updater.versionAvailable", { version: "1.2.0" });
// Output: "Version 1.2.0 is available."
```

### Language Switching

```typescript
import { useTranslation } from "react-i18next";

function LanguageToggle() {
  const { i18n } = useTranslation();

  const toggleLanguage = () => {
    const newLang = i18n.language === "en" ? "zh" : "en";
    i18n.changeLanguage(newLang);
  };

  return <button onClick={toggleLanguage}>Switch Language</button>;
}
```

Use `useTranslation()` directly only when you need `i18n` methods such as `changeLanguage()`.

## Adding New Languages

### 1. Create Translation File

```bash
# Create new locale file
touch src/i18n/locales/ja.json
```

### 2. Add Translations

```json
{
  "app": {
    "title": "Tauriアプリテンプレート",
    "welcome": "Tauri + Reactへようこそ"
  }
}
```

### 3. Register in Configuration

```typescript
// src/i18n/index.ts
import ja from "./locales/ja.json";

i18n.use(initReactI18next).init({
  resources: {
    en: { translation: en },
    zh: { translation: zh },
    ja: { translation: ja }, // Add new language
  },
});
```

## Best Practices

### 1. Namespace Organization

Group related translations by feature:

```json
{
  "auth": {
    "login": "...",
    "logout": "..."
  },
  "profile": {
    "edit": "...",
    "save": "..."
  }
}
```

### 2. Consistent Key Naming

- Use **camelCase** for keys: `checkForUpdates`
- Use **dot notation** for nesting: `updater.checkForUpdates`
- Keep keys descriptive but concise

### 3. Avoid Hardcoded Strings

❌ **Bad:**

```typescript
<button>Check for Updates</button>
```

✅ **Good:**

```typescript
<button>{t("updater.checkForUpdates")}</button>
```

### 4. Maintain Parity

Ensure all locale files have the same structure:

```bash
# Check for missing keys
diff <(jq -S 'keys' src/i18n/locales/en.json) \
     <(jq -S 'keys' src/i18n/locales/zh.json)
```

## Testing

### Test Translation Keys Exist

```typescript
import { renderHook } from "@testing-library/react";
import { useTranslation } from "react-i18next";

test("translation key exists", () => {
  const { result } = renderHook(() => useTranslation());
  expect(result.current.t("app.title")).not.toBe("app.title");
});
```

## Troubleshooting

### Language Not Changing

1. Check localStorage: `localStorage.getItem("i18nextLng")`
2. Clear cache: `localStorage.removeItem("i18nextLng")`
3. Verify language code matches locale file name

### Missing Translations

1. Check console for warnings
2. Verify key path is correct
3. Ensure fallback language has the key

### Interpolation Not Working

```typescript
// ❌ Wrong
t("message", "value");

// ✅ Correct
t("message", { variable: "value" });
```
